
## Суть идеи простыми словами

На любом ядерном объекте постоянно работают тысячи датчиков — температура, давление, вибрация, расход теплоносителя. Большую часть времени всё в норме. Но иногда какой-то параметр начинает вести себя странно — это и есть аномалия.

Проблема не в том, чтобы заметить аномалию когда она уже очевидна. Проблема в том, чтобы заметить её на ранней стадии и объяснить оператору **что именно странно и почему это важно**, а не просто мигнуть красной лампочкой.

Ваша система делает именно это — и при этом честно объясняет каждый вывод.

---

## Что такое аномалия в контексте датчиков

Есть три типа, которые интересны для демо:

**Точечная аномалия** — одно значение резко выбивается из ряда. Температура была 280°C, и вдруг скачок до 340°C. Это легко заметить даже старой системой.

**Контекстуальная аномалия** — значение само по себе нормальное, но ненормальное в данный момент. Температура 290°C — это норма при полной мощности реактора, но аномалия при плановом снижении мощности до 50%. Старые пороговые системы это не замечают.

**Коллективная аномалия** — ни один датчик не вышел за порог, но паттерн нескольких датчиков вместе стал странным. Температура выросла на 2°C, давление упало на 0.3 атм, вибрация насоса изменила частоту на 0.1 Гц — каждое изменение в пределах нормы, но вместе они указывают на начало кавитации насоса. Это самый ценный тип — и именно здесь ваша система даёт преимущество.

---

## Почему "объяснимый" — ключевое слово

Обычный подход на хакатонах: берём нейросеть (LSTM, Autoencoder), обучаем, она говорит "аномалия". Жюри из ядерной отрасли сразу спросит: "А почему она так решила? Можете гарантировать что это не ложная тревога?"

Ответить нечего — нейросеть не объясняет решения.

Ваш подход другой. Вы используете методы, где объяснение встроено в алгоритм:

**Z-score** — самый простой. Для каждого датчика считаем среднее и стандартное отклонение по историческому окну. Если текущее значение отклонилось более чем на 3σ — аномалия. Объяснение автоматическое: "Температура датчика ТЕ-042 отклонилась на 3.7σ от нормы за последние 2 часа."

**CUSUM (Cumulative Sum)** — замечает медленный drift, который Z-score пропускает. Если температура растёт на 0.1°C каждые 10 минут — ни одно отдельное значение не аномально, но накопленная сумма отклонений пересекает порог. Объяснение: "Обнаружен устойчивый тренд роста за 4 часа, накопленное отклонение 2.4°C."

**Isolation Forest** — единственный ML-метод в вашем стеке, но он интерпретируемый. Алгоритм изолирует точки данных случайными разрезами. Аномальные точки изолируются быстрее — им нужно меньше разрезов. Можно извлечь какие признаки (какие датчики) сделали точку лёгкой для изоляции — это и есть объяснение.

Итоговая архитектура детектора: три метода работают параллельно, и аномалия подтверждается только если минимум два из трёх согласны. Это называется ансамблевое голосование и резко снижает ложные срабатывания.

## Как устроен симулятор аномалий

Реальных данных у вас скорее всего не будет, поэтому нужен генератор синтетических временных рядов. Это не слабость проекта — это честный инженерный подход, и жюри это понимают.

Генератор работает так. Берём базовый сигнал — синус с периодом 24 часа (суточный цикл нагрузки реактора) плюс гауссовский шум. Это "нормальная" работа. Затем в случайные моменты времени имплантируем один из сценариев аномалий:

Сценарий "Деградация насоса" — медленный рост вибрации на протяжении 6 часов плюс небольшой рост температуры. Ни один датчик не выходит за порог, но CUSUM это поймает.

Сценарий "Микротрещина трубопровода" — падение давления с одновременным ростом температуры в соседней петле. Изолированно — норма, вместе — аномалия. Isolation Forest это поймает.

Сценарий "Отказ датчика" — резкий скачок одного датчика без изменения соседних. Это ложная тревога, и система должна её правильно классифицировать как "подозрение на отказ датчика, а не реальная аномалия". Это сильный момент для презентации — показывает что система умная, а не просто кричит на каждый шум.

## Что видит оператор на экране

Главный экран — временные ряды всех датчиков. Нормальные участки серые, аномальные подсвечены. Цвет по серьёзности: жёлтый — один метод сработал, оранжевый — два, красный — все три.

Карточка аномалии при клике: название датчика, тип аномалии (точечная / тренд / коллективная), какой метод обнаружил и почему, рекомендуемое действие ("проверить подшипник насоса ЦН-1").

Панель статистики: сколько аномалий за смену, процент ложных тревог на синтетических данных, распределение по типам.

#### Генератор временных рядов

Задача генератора — создать реалистичный поток данных с датчиков и в определённые моменты имплантировать аномалии. Важно понимать: генератор должен возвращать не просто числа, но и метки — где реально была аномалия. Это нужно чтобы потом считать качество детекции.

```python
# generator.py
import numpy as np
from dataclasses import dataclass
from typing import List, Tuple

@dataclass
class SensorReading:
    timestamp: float      # Unix timestamp
    sensor_id: str
    value: float
    is_anomaly: bool      # ground truth для оценки
    anomaly_type: str     # "none", "drift", "spike", "collective"

def generate_baseline(
    n_points: int,
    base_value: float,
    noise_std: float,
    period_hours: float = 24.0
) -> np.ndarray:
    """
    Базовый сигнал: синусоида суточного цикла + гауссовский шум
    """
    t = np.linspace(0, n_points / 60, n_points)  # в часах, 1 точка = 1 минута
    cycle = np.sin(2 * np.pi * t / period_hours)
    noise = np.random.normal(0, noise_std, n_points)
    return base_value + 5 * cycle + noise

def inject_drift(signal: np.ndarray, start: int, duration: int, rate: float) -> Tuple[np.ndarray, np.ndarray]:
    """
    Медленный тренд — деградация насоса или утечка.
    rate: градусов/минуту например
    Возвращает изменённый сигнал и маску аномалий
    """
    result = signal.copy()
    mask = np.zeros(len(signal), dtype=bool)
    for i in range(duration):
        idx = start + i
        if idx < len(result):
            result[idx] += rate * i
            mask[idx] = True
    return result, mask

def inject_spike(signal: np.ndarray, position: int, magnitude: float) -> Tuple[np.ndarray, np.ndarray]:
    """
    Резкий скачок — отказ датчика или гидроудар
    """
    result = signal.copy()
    mask = np.zeros(len(signal), dtype=bool)
    result[position] += magnitude
    mask[position] = True
    return result, mask

def generate_scenario_pump_degradation(n_points: int = 1440) -> List[SensorReading]:
    """
    Сценарий: деградация насоса за 6 часов
    Три датчика: температура, вибрация, давление
    Аномалия коллективная — каждый датчик в пределах нормы,
    но вместе они указывают на проблему
    """
    readings = []
    
    temp = generate_baseline(n_points, base_value=285.0, noise_std=0.5)
    vibration = generate_baseline(n_points, base_value=2.1, noise_std=0.05)
    pressure = generate_baseline(n_points, base_value=15.5, noise_std=0.1)
    
    # Аномалия начинается с точки 800 (примерно 13 часов от начала)
    anomaly_start = 800
    anomaly_duration = 360  # 6 часов
    
    temp, temp_mask = inject_drift(temp, anomaly_start, anomaly_duration, rate=0.003)
    vibration, vib_mask = inject_drift(vibration, anomaly_start, anomaly_duration, rate=0.0005)
    pressure, pres_mask = inject_drift(pressure, anomaly_start, anomaly_duration, rate=-0.0008)
    
    base_time = 1700000000.0
    
    for i in range(n_points):
        t = base_time + i * 60
        readings.append(SensorReading(t, "TE-042", temp[i], bool(temp_mask[i]), "drift" if temp_mask[i] else "none"))
        readings.append(SensorReading(t, "VE-042", vibration[i], bool(vib_mask[i]), "collective" if vib_mask[i] else "none"))
        readings.append(SensorReading(t, "PE-042", pressure[i], bool(pres_mask[i]), "collective" if pres_mask[i] else "none"))
    
    return readings
```

#### Детекторы

Каждый детектор — отдельный класс с единым интерфейсом: принимает массив значений, возвращает массив булевых меток и score для каждой точки. Единый интерфейс важен для ансамбля.

python

```python
# detectors.py
import numpy as np
from dataclasses import dataclass
from typing import List

@dataclass
class DetectionResult:
    is_anomaly: np.ndarray   # bool массив
    score: np.ndarray         # float массив, чем выше тем аномальнее
    explanation: str          # текст для оператора

class ZScoreDetector:
    """
    Скользящее окно: считаем среднее и std за последние window точек.
    Если текущее значение дальше threshold стандартных отклонений — аномалия.
    Объяснение: можем точно сказать на сколько sigma отклонились.
    """
    def __init__(self, window: int = 60, threshold: float = 3.0):
        self.window = window
        self.threshold = threshold
    
    def detect(self, values: np.ndarray, sensor_id: str) -> DetectionResult:
        scores = np.zeros(len(values))
        is_anomaly = np.zeros(len(values), dtype=bool)
        
        for i in range(self.window, len(values)):
            window_data = values[i - self.window:i]
            mean = np.mean(window_data)
            std = np.std(window_data)
            
            if std < 1e-10:  # защита от деления на ноль
                continue
                
            z = abs(values[i] - mean) / std
            scores[i] = z
            is_anomaly[i] = z > self.threshold
        
        # Формируем объяснение для самой аномальной точки
        max_idx = np.argmax(scores)
        max_score = scores[max_idx]
        explanation = (
            f"Датчик {sensor_id}: максимальное отклонение {max_score:.1f}σ "
            f"от скользящего среднего (порог: {self.threshold}σ)"
        ) if max_score > self.threshold else f"Датчик {sensor_id}: норма"
        
        return DetectionResult(is_anomaly, scores, explanation)


class CUSUMDetector:
    """
    Накапливает отклонения от целевого значения.
    Когда накопленная сумма превышает порог — аномалия.
    Ловит медленный drift который Z-score пропускает.
    k: slack parameter, насколько допускаем отклонение без накопления
    h: порог срабатывания
    """
    def __init__(self, k: float = 0.5, h: float = 5.0):
        self.k = k
        self.h = h
    
    def detect(self, values: np.ndarray, sensor_id: str) -> DetectionResult:
        n = len(values)
        scores = np.zeros(n)
        is_anomaly = np.zeros(n, dtype=bool)
        
        # Целевое значение — среднее первых 10% данных (считаем их нормой)
        target = np.mean(values[:max(10, n // 10)])
        std_est = np.std(values[:max(10, n // 10)])
        
        if std_est < 1e-10:
            return DetectionResult(is_anomaly, scores, f"Датчик {sensor_id}: норма")
        
        cusum_pos = 0.0  # накопитель роста
        cusum_neg = 0.0  # накопитель падения
        
        for i in range(n):
            normalized = (values[i] - target) / std_est
            cusum_pos = max(0, cusum_pos + normalized - self.k)
            cusum_neg = max(0, cusum_neg - normalized - self.k)
            scores[i] = max(cusum_pos, cusum_neg)
            is_anomaly[i] = scores[i] > self.h
        
        max_score = np.max(scores)
        # Считаем длину последнего аномального участка
        drift_duration = int(np.sum(is_anomaly))
        
        explanation = (
            f"Датчик {sensor_id}: устойчивый тренд обнаружен, "
            f"накопленное отклонение {max_score:.1f} единиц за {drift_duration} минут"
        ) if max_score > self.h else f"Датчик {sensor_id}: тренд в норме"
        
        return DetectionResult(is_anomaly, scores, explanation)


class IsolationForestDetector:
    """
    Работает на многомерных данных — принимает несколько датчиков сразу.
    Это ключевое отличие: ловит коллективные аномалии.
    Используем sklearn, не пишем с нуля.
    """
    def __init__(self, contamination: float = 0.05, n_estimators: int = 100):
        self.contamination = contamination
        self.n_estimators = n_estimators
    
    def detect(self, multivariate_data: np.ndarray, sensor_ids: List[str]) -> DetectionResult:
        from sklearn.ensemble import IsolationForest
        
        model = IsolationForest(
            contamination=self.contamination,
            n_estimators=self.n_estimators,
            random_state=42
        )
        
        # predict возвращает -1 для аномалий, 1 для нормы
        labels = model.fit_predict(multivariate_data)
        scores_raw = model.score_samples(multivariate_data)
        
        # Инвертируем scores чтобы большее значение = более аномально
        scores = -scores_raw
        is_anomaly = labels == -1
        
        # Для объяснения: какой датчик внёс наибольший вклад
        # Считаем дисперсию каждого датчика в аномальных точках vs норма
        anomaly_points = multivariate_data[is_anomaly]
        normal_points = multivariate_data[~is_anomaly]
        
        if len(anomaly_points) > 0 and len(normal_points) > 0:
            variance_ratio = np.var(anomaly_points, axis=0) / (np.var(normal_points, axis=0) + 1e-10)
            most_contributing = sensor_ids[np.argmax(variance_ratio)]
            explanation = (
                f"Коллективная аномалия по датчикам {', '.join(sensor_ids)}: "
                f"наибольший вклад вносит {most_contributing}. "
                f"Обнаружено {int(np.sum(is_anomaly))} аномальных точек."
            )
        else:
            explanation = f"Датчики {', '.join(sensor_ids)}: норма"
        
        return DetectionResult(is_anomaly, scores, explanation)
```

#### Ансамблевое голосование

python

```python
# ensemble.py
import numpy as np
from dataclasses import dataclass
from typing import List, Dict
from detectors import ZScoreDetector, CUSUMDetector, IsolationForestDetector, DetectionResult

@dataclass
class AnomalyEvent:
    start_index: int
    end_index: int
    severity: str          # "warning", "alert", "critical"
    methods_triggered: List[str]
    explanation: str
    recommendation: str

RECOMMENDATIONS = {
    "drift": "Проверить подшипники и уплотнения насоса. Плановый осмотр в течение 8 часов.",
    "spike": "Вероятен отказ датчика или гидроудар. Проверить датчик и соседние показания.",
    "collective": "Признаки системной проблемы. Требуется осмотр узла в течение 4 часов.",
}

class AnomalyEnsemble:
    def __init__(self):
        self.zscore = ZScoreDetector(window=60, threshold=3.0)
        self.cusum = CUSUMDetector(k=0.5, h=5.0)
        self.iforest = IsolationForestDetector(contamination=0.05)
    
    def analyze(self, sensor_data: Dict[str, np.ndarray]) -> List[AnomalyEvent]:
        """
        sensor_data: словарь {sensor_id: array_of_values}
        """
        results = {}
        sensor_ids = list(sensor_data.keys())
        values_matrix = np.column_stack(list(sensor_data.values()))
        
        # Запускаем все три детектора
        for sid, values in sensor_data.items():
            results[f"zscore_{sid}"] = self.zscore.detect(values, sid)
            results[f"cusum_{sid}"] = self.cusum.detect(values, sid)
        
        iforest_result = self.iforest.detect(values_matrix, sensor_ids)
        
        # Голосование: аномалия подтверждается если минимум 2 метода согласны
        n = len(next(iter(sensor_data.values())))
        vote_matrix = np.zeros(n, dtype=int)
        triggered_methods = [[] for _ in range(n)]
        
        for name, result in results.items():
            for i in range(n):
                if result.is_anomaly[i]:
                    vote_matrix[i] += 1
                    triggered_methods[i].append(name)
        
        for i in range(n):
            if iforest_result.is_anomaly[i]:
                vote_matrix[i] += 1
                triggered_methods[i].append("isolation_forest")
        
        confirmed = vote_matrix >= 2
        
        # Группируем соседние аномальные точки в события
        events = []
        in_event = False
        start = 0
        
        for i in range(n):
            if confirmed[i] and not in_event:
                in_event = True
                start = i
            elif not confirmed[i] and in_event:
                in_event = False
                votes_in_event = vote_matrix[start:i]
                max_votes = int(np.max(votes_in_event))
                
                severity = "warning" if max_votes == 2 else ("alert" if max_votes == 3 else "critical")
                methods = list(set(sum(triggered_methods[start:i], [])))
                
                # Определяем тип аномалии по преобладающему методу
                anomaly_type = "collective"
                if any("zscore" in m for m in methods) and not any("cusum" in m for m in methods):
                    anomaly_type = "spike"
                elif any("cusum" in m for m in methods):
                    anomaly_type = "drift"
                
                # Собираем объяснения от всех методов
                explanations = [results[m].explanation for m in methods if m in results]
                if "isolation_forest" in methods:
                    explanations.append(iforest_result.explanation)
                
                events.append(AnomalyEvent(
                    start_index=start,
                    end_index=i,
                    severity=severity,
                    methods_triggered=methods,
                    explanation=" | ".join(explanations),
                    recommendation=RECOMMENDATIONS.get(anomaly_type, "Требуется осмотр.")
                ))
        
        return events
```

```python
# main.py
from fastapi import FastAPI, WebSocket
from fastapi.middleware.cors import CORSMiddleware
from pydantic import BaseModel
from typing import List, Dict
import numpy as np
import asyncio
import json

from generator import generate_scenario_pump_degradation, SensorReading
from ensemble import AnomalyEnsemble

app = FastAPI()

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_methods=["*"],
    allow_headers=["*"],
)

ensemble = AnomalyEnsemble()

# In-memory store
store: Dict[str, List[SensorReading]] = {}

class AnalyzeRequest(BaseModel):
    scenario: str = "pump_degradation"

@app.post("/api/generate")
def generate_data(request: AnalyzeRequest):
    """Генерирует синтетические данные и сохраняет в памяти"""
    readings = generate_scenario_pump_degradation()
    store["current"] = readings
    
    # Возвращаем в формате удобном для фронта
    sensors = {}
    for r in readings:
        if r.sensor_id not in sensors:
            sensors[r.sensor_id] = []
        sensors[r.sensor_id].append({
            "timestamp": r.timestamp,
            "value": r.value,
            "is_anomaly_ground_truth": r.is_anomaly
        })
    
    return {"status": "ok", "sensors": sensors, "total_points": len(readings)}

@app.post("/api/analyze")
def analyze():
    """Запускает детекцию на текущих данных"""
    if "current" not in store:
        return {"error": "Сначала вызовите /api/generate"}
    
    readings = store["current"]
    
    # Группируем по датчикам
    sensor_data = {}
    for r in readings:
        if r.sensor_id not in sensor_data:
            sensor_data[r.sensor_id] = []
        sensor_data[r.sensor_id].append(r.value)
    
    sensor_arrays = {k: np.array(v) for k, v in sensor_data.items()}
    events = ensemble.analyze(sensor_arrays)
    
    return {
        "events": [
            {
                "start_index": e.start_index,
                "end_index": e.end_index,
                "severity": e.severity,
                "methods_triggered": e.methods_triggered,
                "explanation": e.explanation,
                "recommendation": e.recommendation
            }
            for e in events
        ],
        "total_anomalies": len(events)
    }

@app.websocket("/ws/stream")
async def stream_data(websocket: WebSocket):
    """
    Стримит данные точка за точкой с задержкой —
    имитирует реальный поток с датчиков
    """
    await websocket.accept()
    
    readings = generate_scenario_pump_degradation(n_points=200)
    
    # Группируем по timestamp
    by_time = {}
    for r in readings:
        ts = r.timestamp
        if ts not in by_time:
            by_time[ts] = []
        by_time[ts].append(r)
    
    for ts in sorted(by_time.keys()):
        point = {
            "timestamp": ts,
            "sensors": {
                r.sensor_id: r.value 
                for r in by_time[ts]
            }
        }
        await websocket.send_text(json.dumps(point))
        await asyncio.sleep(0.1)  # 100ms задержка между точками
    
    await websocket.close()
```

### Критичные моменты которые часто ломают демо

Первое — CORS. Он уже прописан в main.py через middleware, но если фронт запущен не на localhost:5173 — добавьте этот порт в allow_origins.

Второе — синхронизация форматов. Генератор возвращает объект где ключи — sensor_id, а значения — массивы точек. Если фронт ожидает что-то другое — демо упадёт. Проверьте это первым делом при интеграции.

Третье — WebSocket в React. useEffect должен возвращать функцию cleanup которая закрывает соединение, иначе при перерендере будут множественные соединения и хаос в данных.

Четвёртое — Isolation Forest требует минимум ~50 точек для нормальной работы. На меньших данных он будет давать мусор. Убедитесь что генератор даёт достаточно данных перед запуском анализа.
